## 1. 概要

FITOM_X は 16 個の MIDI チャンネルを持ち、各チャンネルは独立して「メロディチャンネル(通常の音源チャンネル)」または「リズムチャンネル(パーカッション専用チャンネル)」のいずれかとして動作します。

General MIDI(GM)のようにチャンネル10が固定でドラム用というルールはなく、どのチャンネルもコントロールチェンジによって動的にメロディ/リズムを切り替えられます(詳細は「4.1 チャンネルの役割切替」参照)。

各チャンネルが実際にどの音源(チップ)・どの音色を鳴らすかは、バンクセレクト(CC#0・CC#32)とプログラムチェンジの組み合わせ、およびロードされているプロファイル(音色データ)の内容によって決まります。

## 2. チャンネルボイスメッセージ

### 2.1 ノートオン / ノートオフ

ノートナンバーは 0–127、ベロシティは 1–127 を受け付けます。ノートオンでベロシティ=0を送信した場合はノートオフとして扱われます(標準MIDI仕様通り)。

メロディチャンネルでは、同時に鳴らせるノート数はチャンネルごとに最大16ですが、実際の上限はさらに「5. モノ/ポリモードとボイス数上限」で説明するボイス数上限、および使用中の音色が要求するデバイスの発音数によって制限されます。上限を超えてノートオンを受信すると、最も古く発音を開始したノートから自動的に停止(ボイススティール)されて新しいノートに割り当てられます。

リズムチャンネルでは、ノートナンバーごとに個別の楽器(ドラム/パーカッション音)が独立して発音します(詳細は「4. チャンネルの役割とリズムチャンネル」参照)。

### 2.2 バンクセレクト / プログラムチェンジ

音色の選択は、CC#0(バンクセレクトMSB)・CC#32(バンクセレクトLSB)・プログラムチェンジの3つの組み合わせで行います。CC#0の値によって、以下の3つの動作モードのいずれかになります。

#### ① 通常モード(CC#0 = 0)

プロファイルに定義されたバンクから音色を選択する、標準的なモードです。

- CC#32(バンクセレクトLSB) : バンク番号
- プログラムチェンジ : バンク内のプログラム番号(0–127)

該当するバンク・プログラムが見つからない場合、直前まで使用していた音色が維持されます(無音にはなりません)。

#### ② 直接デバイス選択モード(CC#0 = 1–111)

特定のチップ(音源デバイス)の音色バンクを直接指定するモードです。プロファイルの構成によらず、対応するチップが接続されていれば音色を鳴らすことができます。

- CC#0 : 対象チップの種別(下表「直接デバイス選択値 一覧」参照)
- CC#32 : そのチップ用バンクのインデックス番号
- プログラムチェンジ : バンク内のプログラム番号

#### ③ チャンネル役割切替(CC#0 = 120 / 121)

CC#0 に 120(0x78)を送ると、そのチャンネルはリズムチャンネルに切り替わります。121(0x79)を送るとメロディチャンネルに戻ります。切替と同時にプログラム0番が選択された状態になります。詳細は「4.1 チャンネルの役割切替」を参照してください。

CC#0 = 112(0x70)は、内蔵リズム音源(ROM固定のドラム音を持つ一部チップ)専用バンクへのアクセスに使われる特別な値です。CC#32で対象チップ、プログラムチェンジで内蔵楽器の番号を指定します。対応するプロファイルが構成されている場合のみ使用できます。

### 2.4 ピッチベンド

14bit分解能のピッチベンドを受け付けます。ベンドの可変幅(センシティビティ)はRPN#0で半音単位で設定します(デフォルト:±2半音)。センシティビティの小数点以下(セント)成分は使用されません。

### 2.5 アフタータッチ

ポリフォニックキープレッシャー、チャンネルプレッシャーのいずれも受信しません(受信しても無視されます)。

## 3. RPN / NRPN

RPN・NRPNは、CC#100/101(RPN)またはCC#98/99(NRPN)でパラメータ番号を選択したのち、CC#6(データエントリーMSB)・CC#38(データエントリーLSB)で値を書き込みます。CC#96(データインクリメント)・CC#97(データデクリメント)で、選択中のパラメータの値を1ステップ単位で増減させることもできます。

### 3.1 RPN(レジスタードパラメータナンバー)

| RPN (MSB,LSB) | 名称 | 内容 |
|---|---|---|
| 0, 0 | ピッチベンドセンシティビティ | データエントリーMSBの値がそのまま半音数になります(0–127半音)。LSB(セント)は使用されません。CC#96/97は1半音単位で増減します。 |
| 0, 1 | チャンネルファインチューニング | セント単位の微調整(14bit値、中央値でピッチ変化なし)。CC#96/97は、データエントリーMSBを1つ送った場合と同じ幅(値にして128)で増減します。 |
| 0, 2 | チャンネルコースチューニング | 半音単位の粗調整(データエントリーMSBの値、中央値=64でピッチ変化なし)。CC#96/97は1半音単位で増減します。 |
| 0, 5 | モジュレーションデプスレンジ | CC#1(モジュレーション)によるビブラートの最大デプスをセント単位で設定します(データエントリーMSBの値を使用)。CC#96/97は1セント単位で増減します。 |
| 127, 127 | RPN NULL | RPN/NRPNの選択状態を解除します(以降のデータエントリー、データインクリメント/デクリメントは無視されます)。 |

上記以外のRPN番号は受信しても無視されます。

### 3.2 NRPN(ノンレジスタードパラメータナンバー)

#### メロディチャンネル共通

| NRPN (MSB,LSB) | 名称 | 内容 |
|---|---|---|
| 96, 1 (0x60, 0x01) | 物理チャンネル固定 | 以下の条件をすべて満たす場合のみ有効になります。①そのチャンネルがモノフォニックモード(5.1節、CC#126でM=1)であること、②直接デバイス選択モード(2.2節②)で有効なチップを選択していること、③指定したチャンネル番号がそのチップの発音チャンネル数(複数チップを束ねて1つとして扱う構成の場合は合算後の数)の範囲内であること。条件を満たすと、DVA(自動チャンネル割り当て)を経由せず、常に指定した発音チャンネルへ割り当てます。値はデータエントリーMSB(0–127)で指定します(LSBは使用しません)。この固定状態は、新たにこのNRPNを受信したとき・プログラムチェンジ・バンクセレクト(MSB/LSBいずれも)のいずれかを受信すると解除されます(条件を再度満たせば同時に再設定されます)。特殊な用途(外部シーケンサーとの連携等)向けの設定です。 |
| 96, 2 (0x60, 0x02) | パフォーマンスバンクセレクト | 次に送る96,3(パフォーマンスプログラムセレクト)とセットで使います。このNRPN単体では何も変化しません(値を一時的に保持するだけです)。 |
| 96, 3 (0x60, 0x03) | パフォーマンスプログラムセレクト | 受信すると、現在このチャンネルが使っているパフォーマンスパッチ(演奏特性データ)を、直前に受信した96,2の値をバンク番号、この96,3自身のデータエントリーMSBの値をプログラム番号として、明示的に差し替えます。現在選択中の音色(トーン)自体は変更されません。指定したバンク/プログラムが見つからない場合は、音色本来のパフォーマンスパッチにソフトフォールバックします(無音にはなりません)。**次にプログラムチェンジを受信するまで有効**です。 |
| 97, 0–7 / 8–15 / 16–23 / 24–31 | ToneLayerオーバーライド(レイヤー0〜3) | 次節「3.3 ToneLayerオーバーライド(NRPN 97)」参照。 |

#### リズムチャンネル専用(GM2ドラムパート単位パラメータ)

リズムチャンネルでは、NRPNのLSBにノートナンバー(0–127)を指定することで、ドラム楽器ごとに個別のパラメータを調整できます(General MIDI 2 のDrum Instrument系NRPNに準拠)。

| NRPN MSB | 名称 | 内容 |
|---|---|---|
| 24 (0x18) | ドラムインストゥルメントピッチ | 指定ノートの音程オフセット。データエントリーMSBの値-64が半音単位のオフセットになります(64=変化なし、範囲-64〜+63半音)。CC#96/97は1半音単位で増減します。 |
| 26 (0x1A) | ドラムインストゥルメントレベル | 指定ノートのベロシティオフセット。データエントリーMSBの値-64がオフセットになります(64=変化なし)。CC#96/97は1単位で増減します。 |
| 28 (0x1C) | ドラムインストゥルメントパン | 指定ノートのパンオフセット。データエントリーMSBの値-64がオフセットになります(64=変化なし)。CC#96/97は1単位で増減します。 |

NRPN LSBには、調整したいドラム楽器のノートナンバーをそのまま指定します。リズムチャンネルではRPNは使用しません(受信しても無視されます)。

### 3.3 ToneLayerオーバーライド(NRPN 97)

**通常モード(2.2節①)で音色を選択しているメロディチャンネルに限り**、その音色を構成するレイヤー(1つの音色が複数のチップを重ねて鳴らす仕組み)のパラメータを、演奏中にNRPNで上書きできます。直接デバイス選択モード(2.2節②)のチャンネルでは、これらのNRPNは受信しても何も起こりません。

対象レイヤーは0〜3の4つで、レイヤーごとに8個のパラメータを持ちます。NRPN LSB = レイヤー番号×8 + パラメータ番号(0〜7)です。

| パラメータ番号 | 名称 | 内容 |
|---|---|---|
| 0 | 音源チップ選択 | このレイヤーが使うチップの種別(2.2節②の直接デバイス選択値と同じ体系)。**保持のみ**(単体では反映されません。パラメータ2とセットで使います)。 |
| 1 | 音色バンク選択 | チップ用バンクのインデックス。**保持のみ**(同上)。 |
| 2 | 音色プログラム選択 | バンク内のプログラム番号。**受信した瞬間に、直前に受信したパラメータ0・1とセットでこのレイヤーの音を丸ごと差し替えます**(音源チップそのものを含めた置き換えです)。 |
| 3 | トランスポーズ | 半音単位のオフセット(データエントリーMSBの値-64、64=変化なし)。**次のノートオンから即座に反映**されます。 |
| 4 | 音域(下限) | このレイヤーが発音する最低ノートナンバー(0–127)。同上。 |
| 5 | 音域(上限) | このレイヤーが発音する最高ノートナンバー(0–127)。同上。 |
| 6 | ボリュームオフセット | データエントリーMSBの値-64(64=変化なし)。同上。 |
| 7 | パンオフセット | データエントリーMSBの値-64(64=変化なし)。同上。 |

レイヤー0は`(97, 0)`〜`(97, 7)`、レイヤー1は`(97, 8)`〜`(97, 15)`、レイヤー2は`(97, 16)`〜`(97, 23)`、レイヤー3は`(97, 24)`〜`(97, 31)`です。

パラメータ0・1・2(音源チップ/バンク/プログラム選択)と、パラメータ3〜7(トランスポーズ・音域・ボリューム・パンオフセット)は、それぞれ独立して機能します。たとえば音色そのものは差し替えず、特定レイヤーのパン位置だけを演奏中に動かす、といった使い方もできます。

音色データが元々定義していないレイヤー番号(例:2レイヤー構成の音色に対してレイヤー2や3)に対しても、パラメータ0/1/2を使って新たにチップ・音色を割り当てることができます(レイヤーを追加する形になります)。

**すべてのパラメータは、次にプログラムチェンジを受信するまで有効**です。「次のノートオンから反映」と記載のあるパラメータは、既に発音中のノートには影響しません(新しく鳴らすノートから反映されます)。

## 4. チャンネルの役割とリズムチャンネル

### 4.1 チャンネルの役割切替

各MIDIチャンネルは、以下のバンクセレクトMSB(CC#0)の特殊な値によって、メロディチャンネルとリズムチャンネルを動的に切り替えられます。

- CC#0 = 120(0x78) を送ると、そのチャンネルはリズムチャンネルになります。
- CC#0 = 121(0x79) を送ると、そのチャンネルはメロディチャンネルに戻ります。

切替が発生すると、それまで発音していたノートはすべて停止し、プログラム0番が選択された状態から始まります。切替前のボリューム・パン等の設定は引き継がれません。

この2つの値は役割切替専用として予約されているため、通常のバンク選択(2.2節①)や直接デバイス選択(同②)には使用できません。

### 4.2 リズムチャンネルの動作

リズムチャンネルでは、受信したノートナンバーごとに独立した楽器(ドラム/パーカッション音)が割り当てられます。どのノートナンバーにどの楽器が割り当てられるかは、プログラムチェンジで選択されるドラムキットの構成によって決まります。

リズムチャンネルは、メロディチャンネルと異なる以下の特性を持ちます。

- バンクセレクト(CC#0・CC#32)は使用しません。音色の切り替えはプログラムチェンジ(ドラムキットの選択)のみで行います。
- ボリューム(CC#7)は反映されますが、パン・エクスプレッション・ピッチベンド・サステイン・モジュレーション等のコントロールチェンジは反映されません。
- アフタータッチ、ポルタメント、レガート、ソステヌート、モノ/ポリモード切替は非対応です。
- 楽器によっては「ゲートタイム」が設定されている場合があり、その場合はノートオフを受信しても、設定された時間が経過するまで自動的には停止しません(ノートオフは無視されます)。ゲートタイムが設定されていない楽器は、通常通りノートオフで停止します。
- 一部のドラムキットでは、特定のノート同士が「チョークグループ」としてまとめられている場合があります。この場合、グループ内のいずれかのノートを鳴らすと、同じグループの他の発音中のノートは自動的に停止します(例:オープンハイハットを鳴らすとクローズドハイハットが止まる、等)。

楽器ごとの音程・ベロシティ・パンの微調整は、NRPN(3.2節「リズムチャンネル専用」)で行えます。

## 5. モノ/ポリモードとボイス数上限

メロディチャンネルは、CC#126(モノモードオン)・CC#127(ポリモードオン)によって、動作モードとボイス数の上限を切り替えられます。

### 5.1 モノモードオン(CC#126)、値 M = 1

そのチャンネルは真のモノフォニックとして動作します。常に1音のみが発音され、新しいノートオンを受けると直前のノートが自動的に切り替わります(ノート単位のスティール)。このモードでのみ、ポルタメント(CC#5・#65・#84)およびレガート(CC#68)が効果を持ちます。

### 5.2 モノモードオン(CC#126)、値 M = 0 または M ≧ 2

そのチャンネルはポリフォニックのまま、同時発音数(ボイス数)の上限が設定されます。

- M = 0 の場合:上限は、選択中の音色が使用するデバイスから自動的に算出された値になります。
- M ≧ 2 の場合:上限はM(最大16)になります。

上限に達した状態でさらにノートオンを受信すると、**発音を開始した順番が最も古いノート**が自動的に停止し、新しいノートに置き換わります(和音の中で最初に押した音から順に消えていきます)。このモードでは、ポルタメント・レガートは効果を持ちません(通常のポリフォニック動作と同じくノートオンのたびにエンベロープが再トリガーされます)。

このボイス数上限は、プログラムチェンジで音色を切り替えても保持されます。

### 5.3 ポリモードオン(CC#127)

モノモード・ボイス数上限の指定を解除し、通常のポリフォニック動作(上限は選択中の音色から自動算出)に戻します。

## 6. ハードウェアLFO(CC#14/15、OPM/OPZ等)

OPM/OPZ等、チップ自体に内蔵のLFO(ハードウェアLFO)を持つ音源向けの、非標準の拡張コントロールチェンジです。GM/GM2規格には存在しない、FITOM_X独自の割り当てです。

- **CC#14(デプス)**:チップ内蔵LFOの変調量を設定します。
- **CC#15(レート)**:チップ内蔵LFOの速さを設定します。

チップ内蔵LFOは1チップにつき1系統のみのハードウェアリソースです。CC#14/15で設定したデプス/レートは、そのチップを使用している全チャンネルに共通で適用されます(実機のハードウェア構成上、チャンネルごとに別々のデプス/レートを持たせることはできません)。複数のMIDIチャンネルが同じチップを共有している場合、最後に受信したCC#14/15の値が、そのチップ全体に反映されます。

ある音がハードウェアLFOによってピッチ方向(ビブラート)に揺れるか、音量方向(トレモロ)に揺れるかは、CC#14/15ではなく、**発音している音色自体の設定**(AMS/PMSと呼ばれる感度パラメータ)によって決まります。音色がどちらの感度も持たない場合、CC#14/15を送っても聴感上の変化はありません。

チップが複数の物理チップにまたがる構成(スパニング)になっている場合、物理チップごとに独立したハードウェアLFOリソースを持つため、そのチャンネルが実際に発音している物理チップに対してのみCC#14/15が反映されます。

## 7. ソフトウェアLFOパラメータの上書き(CC#76/77/78)

音色データが内蔵する、音色固有のソフトウェアLFO(ビブラート)のRate/Depth/Delayを、演奏中にリアルタイムで上書きするための、非標準の拡張コントロールチェンジです(GM2のSound Controller 7/8/9の番号を流用していますが、動作はGM2規格の「オフセット」方式ではなく、値をそのまま置き換える「上書き」方式です)。

| CC# | 名称 | 内容 |
|---|---|---|
| 76 | ソフトウェアLFO レート上書き | 0–127。音色データが持つレート値をそのまま置き換えます。 |
| 77 | ソフトウェアLFO デプス上書き | 0–127を±1200セント(1オクターブ)の範囲に対応させ、音色データが持つデプス値を置き換えます(127で最大デプス)。 |
| 78 | ソフトウェアLFO ディレイ上書き | 0–127。発音開始からLFOがかかり始めるまでの時間(20msきざみ)を、音色データが持つ値の代わりに使います。 |

デプス(CC#77)は演奏中のノートにも即座に反映されます。レート(CC#76)とディレイ(CC#78)は、LFOが(再)始動するタイミング、つまり**次に鳴らすノートオンから**反映されます(すでに鳴っているノートのLFOの速さやディレイを、鳴っている最中に変えることはできません)。

このソフトウェアLFOは、音色データ自体に「音色固有のLFOを使う」設定(レートが0でない)が入っている場合にのみ動作します。音色データ側でこの設定が無効(CC#1駆動のモジュレーションホイール方式)になっている場合でも、CC#76でレートを明示的に指定すると、そのチャンネルは音色固有LFO方式に切り替わります。

この上書きは、リセットオールコントローラー(CC#121)で解除され、音色データ本来の値に戻ります。

## 8. システムエクスクルーシブメッセージ

Universal Real Time SysEx(`F0 7F <デバイスID> ...`)のうち、以下のメッセージに対応しています。デバイスIDの値は判定されず、常に自分宛のメッセージとして扱われます。Universal Non-Real Time SysEx(`F0 7E ...`)、および他社製品のシステムエクスクルーシブメッセージには対応していません。

| メッセージ | 内容 |
|---|---|
| マスターボリューム<br>`F0 7F <dev> 04 01 ll mm F7` | システム全体の音量を設定します(mmの値、0–127、7bit精度で反映)。 |
| マスターファインチューニング<br>`F0 7F <dev> 04 03 ll mm F7` | システム全体のピッチを±100セントの範囲で微調整します(14bit値、中央値でピッチ変化なし)。 |
| マスターコースチューニング<br>`F0 7F <dev> 04 04 ll mm F7` | システム全体のピッチを半音単位で調整します(mmの値、中央値=64でピッチ変化なし、範囲±64半音)。 |
| スケール/オクターブチューニング(1byte形式)<br>`F0 7F <dev> 08 08 tt cc [12バイト] F7` | 12半音それぞれのチューニングオフセットを設定します(各1バイト、中央値=64でピッチ変化なし)。適用対象チャンネルの指定(チャンネルマスク)は読み飛ばされ、常に全チャンネルに適用されます。 |

上記以外のシステムエクスクルーシブメッセージは受信しても無視されます。

## 8.1 パッチパラメータオーバーライド(プライベートSysEx、非標準)

FM音源チップの個別パラメータ(FB/ALG/AMS/PMS、各オペレータのAR/DR/SR/RR/SL/TL/MUL/DT等、以下「HwPatch」)や、ビブラート・トレモロ等の演奏特性パラメータ(以下「SwPatch」)を、演奏中にリアルタイムで、または特定のプリセット音色そのものを、細かく上書きするための独自拡張です。GM/GM2規格には無い、FITOM_X独自のプライベートSysExです。

```
F0 00 48 01 <sub-cmd> <target-type> <target-addr...> <layer> <JSONバイト列...> F7
```

| フィールド | 内容 |
|---|---|
| `sub-cmd` | `0x01` = HwPatch(音色そのもののパラメータ) / `0x02` = SwPatch(演奏特性パラメータ)。 |
| `target-type` | `0x00` = MIDIチャンネル(演奏中の音を一時的に上書き) / `0x01` = プリセットバンク(該当バンク/プログラムのデータそのものを直接編集)。 |
| `target-addr` | `sub-cmd`と`target-type`の組み合わせにより異なります(下記)。 |
| `layer` | 対象のレイヤー番号(0起算。1つの音色が複数のチップを重ねて鳴らす「レイヤー」を持つ場合に、どのレイヤーを対象にするかを指定します)。`target-type=0x01`のときは無視されます(バンク/プログラムの指定自体が単一のパッチを一意に指すため)。 |
| JSON | オーバーライドしたいパラメータのみを含むJSONオブジェクト(ASCII)。 |

**`target-addr`の内容**

| sub-cmd | target-type | バイト数 | 内容 |
|---|---|---|---|
| 0x01(HwPatch) | 0x00(チャンネル) | 1byte | MIDIチャンネル番号(0–15) |
| 0x01(HwPatch) | 0x01(バンク) | 3byte | 直接デバイス選択値(2.2節②のCC#0相当)/ HwBankインデックス(CC#32相当) / プログラム番号(プログラムチェンジ相当) |
| 0x02(SwPatch) | 0x00(チャンネル) | 1byte | MIDIチャンネル番号(0–15) |
| 0x02(SwPatch) | 0x01(バンク) | 2byte | SwBankインデックス / SwProg番号 |

SwPatch(演奏特性パッチ)はチップの種類に依存しない共通のバンク空間を持つため、HwPatchと異なり直接デバイス選択値を指定する必要がありません。

**JSONの書式**:HwBank / SwBank(音色バンク)ファイルで使われているものと同じキー名を使います。

- HwPatch(`sub-cmd=0x01`)の例:`{"FB":5,"ALG":3,"ops":[{"AR":31,"TL":10},{},null,null]}`
- SwPatch(`sub-cmd=0x02`)の例:`{"sw":{"LFR":10,"LFD":5,"depth_cents":300},"ops":[{"VAR":20},null],"fine_transpose":50}`

`ops`はいずれも0〜4要素の可変長配列で、`null`または`{}`を指定したオペレータは変更されません(現在値を維持)。`"name"`・`"id"`・`"builtin"`はこの経路では指定できません(無視されます)。

**動作**

- `target-type=0x00`(チャンネル):**次にプログラムチェンジを受信するまで有効**です。空のJSONオブジェクト`{}`を送ると、そのレイヤーのオーバーライドを解除し、本来のパラメータに戻します。
  - HwPatch:以後のノートオン、および現在発音中のノートの両方に即座に反映されます。
  - SwPatch:**以後のノートオンから反映されます**(現在発音中のノートには遡って反映されません)。
- `target-type=0x01`(プリセットバンク):該当するバンク/プログラムのデータをメモリ上で直接書き換えます。以後、そのバンク/プログラムを新たに選択するすべてのチャンネルに影響します(発音中のノートには影響しません)。**この変更はメモリ上のみで、ディスク上のファイルへは保存されません**(再起動すると失われます)。存在しないバンク/プログラムを指定した場合は無視されます。

構文の誤ったJSONを送った場合は、メッセージ全体が無視されます(一部だけ適用されることはありません)。

## 9. 対応していないメッセージ・既知の制限

以下のメッセージ・機能は、現在のバージョンでは対応していません。

- ポリフォニックキープレッシャー、チャンネルプレッシャー(アフタータッチ)
- オムニオン/オフ(CC#124–125)
- ローカルコントロール(CC#122)
- ブレスコントローラー(CC#2)、フットコントローラー(CC#4) — 受信しても対応する効果がありません
- ソングポジションポインタ、ソングセレクト、チューンリクエスト
- MIDIスタート/コンティニュー/ストップ、システムリセット
- GM System On/Off、Identity Request等のUniversal Non-Real Time SysEx
- メーカー固有のシステムエクスクルーシブメッセージ

また、以下の点にご注意ください。

- チャンネルは常にオムニオフとして動作します(オムニオンへの切替は不可)。
- リズムチャンネルはモノ/ポリモード切替(CC#126/127)、ポルタメント、レガート、ソステヌート、ハードウェア/ソフトウェアLFO上書きの対象外です。
- CC#14/15(ハードウェアLFO)は、対応するチップ(OPM/OPZ等)以外では効果がありません。
