# FITOM_X SwPatch リファレンス

「SwPatch」は、チップのレジスタに直接書き込まれる合成パラメータ(HwPatch)とは別に、ソフトウェア側で処理される演奏特性のデータです。ビブラート(ピッチのゆらぎ)、トレモロ(音量のゆらぎ)、ベロシティ(鍵盤を弾く強さ)への反応の付け方などを指定します。チップの種類に依存しない、共通のバンク形式です。

音色そのものの合成パラメータは[HwPatch リファレンス](./hwpatch-reference.md)、ネイティブパッチ全体の構成は[ネイティブパッチ リファレンス](./native-patch-reference.md)を参照してください。MIDI経由でこれらのパラメータを演奏中に上書きする方法は[MIDIメッセージリファレンスマニュアルの「8.1 パッチパラメータオーバーライド」](./midi-message-reference.md#81-パッチパラメータオーバーライドプライベートsysex非標準)、および専用のコントロールチェンジ(CC#76/77/78)については[同マニュアルの「7. ソフトウェアLFOパラメータの上書き」](./midi-message-reference.md#7-ソフトウェアlfoパラメータの上書きcc767778)を参照してください。

---

## 1. 全体構成

```json
{
  "name": "Strings Perf",
  "patches": [
    {
      "prog": 0,
      "name": "Slow Vibrato",
      "sw": {
        "LWF": 6, "LFS": 0, "LFM": 0,
        "LFD": 20, "LFR": 30, "LFI": 10,
        "depth_cents": 50
      },
      "ops": [
        { "VTL": 20, "SLW": 6, "SLR": 40, "SLD": 10 },
        { "VTL": 20 },
        { "VTL": 20 },
        { "VTL": 20 }
      ],
      "fine_transpose": 0
    }
  ]
}
```

1つのSwPatchは、チャンネル単位のビブラート設定(`sw`)、オペレータ単位のベロシティ感度・トレモロ設定(`ops`、常に4要素)、微調整用のトランスポーズ(`fine_transpose`)を持ちます。

## 2. チャンネル単位:ビブラート(`sw`)

音程全体を周期的に揺らす、ピッチLFO(ビブラート)の設定です。

| フィールド | 範囲 | 内容 |
|---|---|---|
| `LWF` | 0–6 | 波形。0=上昇のこぎり波/1=矩形波/2=三角波/3=サンプル&ホールド/4=下降のこぎり波/5=デルタ波/6=サイン波 |
| `LFS` | 0/1 | ノートオンのたびに波形の位相をリセットするか(0=しない/1=リセットする) |
| `LFM` | 0–2 | 動作モード。0=繰り返し/1=1周期だけ再生して最後の値を保持/2=1周期だけ再生してゼロに戻る |
| `LFD` | 0–127 | ディレイ(発音開始からLFOがかかり始めるまでの時間、20msきざみ) |
| `LFR` | 0–127 | レート(速さ)。0=LFO無効。1〜127は緩やかな加速度で0.5Hz〜50Hzに対応します(数値を上げるほど速くなります)。 |
| `LFI` | 0–127 | フェードインの速さ(0=最初から最大デプスで開始) |
| `depth_cents` | -1200–+1200 | 深さ(セント単位)。0=変調なし。±1200で1オクターブ分の振れ幅になります。通常のビブラートは数十〜百数十セント程度が目安です。 |

**`LFR=0`(レート未設定)の音色は、代わりにMIDIのCC#1(モジュレーションホイール)でビブラートの深さを演奏時にコントロールする方式になります。** `LFR`に0より大きい値を指定すると、音色固有のLFOとして常に動作し、CC#1は無視されるようになります。

## 3. オペレータ単位パラメータ(`ops`、4要素固定)

各オペレータに対応する4つの要素を持つ配列です。要素0〜3が、そのHwPatchのオペレータ0〜3に対応します。

### ベロシティ感度

鍵盤を弾く強さ(ベロシティ)に応じて、音色のパラメータをどれだけ変化させるかを指定します。弱く弾くほど、音量が小さく・アタックが遅く・音の抜けが柔らかくなる、といった表現に使います。

| フィールド | 範囲 | 内容 |
|---|---|---|
| `VTL` | 0–127 | ベロシティ/エクスプレッション→音量(TL)感度。弱く弾く、またはエクスプレッション(CC#11)が低いほど音量が下がります(どちらか低い方の値が採用されます)。0なら両方とも音量に影響しません。中〜高域はなだらかに、極端に弱い値でのみ大きく減衰する知覚カーブになっています。 |
| `VAR` | 0–127 | ベロシティ→アタックレート感度。弱く弾くほどアタックが遅くなります。 |
| `VDR` | 0–127 | ベロシティ→ディケイレート感度。 |
| `VSL` | 0–127 | ベロシティ→サステインレベル感度。 |
| `VSR` | 0–127 | ベロシティ→サステインレート感度。 |
| `VRR` | 0–127 | ベロシティ→リリースレート感度。 |
| `VLD` | 0–127 | (予約、現在のバージョンでは効果がありません) |
| `VLR` | 0–127 | (予約、現在のバージョンでは効果がありません) |

ベロシティ感度は、和音の中で実際に音を出す(キャリアの)オペレータにのみ適用されます。

### トレモロ(オペレータ単位の音量LFO)

各オペレータの音量を個別に周期的に揺らす設定です。「`sw`」のビブラートと同じ考え方で、対象がピッチではなく音量になります。

| フィールド | 範囲 | 内容 |
|---|---|---|
| `SLW` | 0–6 | 波形(`LWF`と同じ選択肢) |
| `SLS` | 0/1 | ノートオンでリセットするか |
| `SLM` | 0–2 | 動作モード(`LFM`と同じ選択肢) |
| `SLD` | 0–127 | 深さ。0〜63が正方向、64〜127が負方向(-64〜-1)に対応します。 |
| `SLY` | 0–127 | ディレイ(20msきざみ) |
| `SLR` | 0–127 | レート(`LFR`と同じ、0=LFO無効、1〜127で0.5Hz〜50Hz) |
| `SLI` | 0–127 | フェードインの速さ |

## 4. `fine_transpose`

| フィールド | 範囲 | 内容 |
|---|---|---|
| `fine_transpose` | -1200–+1200 | セント単位の微調整トランスポーズ。ネイティブパッチのToneLayerが持つ`transpose`(半音単位)とは独立して働き、両方指定されている場合は加算されます。 |

## 5. 演奏中の上書き

MIDIでは、チャンネル単位で以下の上書きができます。

- **CC#76/77/78**:`sw`のレート・デプス・ディレイを演奏時に一時的に上書きします(3つとも独立、次のノートオンから反映)。
- **プライベートSysEx(sub-cmd=0x02)**:このドキュメントで説明したすべてのフィールド(`sw`・`ops`・`fine_transpose`)を、JSON形式で部分的に上書きできます。発音中のノートには遡って反映されません(次のノートオンから反映されます)。

詳細は[MIDIメッセージリファレンスマニュアル](./midi-message-reference.md)を参照してください。
