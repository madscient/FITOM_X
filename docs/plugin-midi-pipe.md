# 内部用MIDIパイプ インターフェース仕様

**対象**: 外部プロセス(パッチエディタ等の別プロジェクト)からFITOM_Xへ、ローカルのみでMIDIメッセージを送るためのインターフェース
**実装**: `backends/midi_pipe/`(FITOM_X本体側、`fitom_midi_pipe` バックエンドDLL)
**想定する利用者**: 別プロジェクトとして実装されるパッチエディタ

---

## 1. 目的

パッチエディタは、オフライン時(FITOM_Xが起動していない状態)はプロファイル・バンクファイル(JSON)を直接読み書きして動作します。FITOM_Xが起動している間は、**編集中のパッチをその場で試聴**できるよう、通常のMIDI入力と同じ経路でFITOM_X本体へパッチデータ(パラメータオーバーライド)やノートオン/オフを送ります。

このインターフェースは、FITOM_X側の既存の「MIDIバックエンドプラグイン」の仕組み(`plugin_sdk/include/fitom/IMidiPlugin.h`)を使った、名前付きパイプ(Windows)/UNIXドメインソケット(Linux/macOS)経由の実装です。FITOM_X本体からは、他のMIDIキーボード等と全く同じ「MIDI入力の1つ」として扱われます。

---

## 2. 接続先

| プラットフォーム | 接続先 | 種別 |
|---|---|---|
| Windows | `\\.\pipe\FITOM_X_MIDI` | 名前付きパイプ(バイトモード) |
| Linux / macOS | `/tmp/fitom_x_midi.sock` | UNIXドメインソケット(`SOCK_STREAM`) |

接続先は固定です。パッチエディタ側でプロファイルファイルを読む必要はありません。

**接続の性質**:
- FITOM_X側は起動時にこのパイプ/ソケットをリッスン状態にします。パッチエディタは、FITOM_Xが起動しているタイミングでのみ接続でき、起動していない場合は接続に失敗します(オフライン動作時はこれを許容し、静かに「試聴不可」として扱ってください)。
- 同時に接続できるクライアントは1つです。
- 接続が切れても、FITOM_X側は次の接続を自動的に待ち受けます(再接続可能)。
- 送信専用です。FITOM_Xからパッチエディタへの返信(MIDI Out)は現状実装されていません。応答が必要な処理(例:デバイス一覧の取得)は、このパイプとは別の手段(将来的な検討事項)が必要です。

---

## 3. ワイヤーフォーマット

生のMIDI 1.0バイト列をそのまま書き込むだけです。特別なフレーミング(長さプレフィックス等)は不要です。ステータスバイト(最上位ビットが1)によって自己区切りされるMIDIの性質をそのまま利用しています。1回の書き込みで複数メッセージをまとめて送っても、逆に1メッセージを複数回の書き込みに分割して送っても構いません(FITOM_X側で受信バイトを継続的に蓄積・解釈します)。

---

## 4. FITOM_X側の設定

FITOM_Xのプロファイル(`*.profile.json`)に、以下を追加してビルドしてください。

```json
{
  "midi_backend": { "dll": "fitom_midi_pipe" },
  "midi_inputs": ["FITOM Internal Pipe"]
}
```

`midi_inputs`配列の中でのインデックス(この例では0番目)が、そのままMPU(16chの処理単位)番号になります。パッチエディタが使うMIDIチャンネルは、このMPU内の任意の1chで構いません(後述のプログラムチェンジで直接モードを選択するため、他のチャンネルと衝突する心配はありません)。

CMakeビルド時は `-DFITOM_BUILD_BACKEND_MIDI_PIPE=ON` を指定してください(既定はOFF)。

---

## 5. 送信するメッセージの組み立て方

### 5.1 試聴対象の音源を選ぶ(直接デバイス選択モード)

編集中のHwPatch/SwPatchと同じ音源を鳴らすため、まず直接デバイス選択モードでチップ・バンク・プログラムを指定します(詳細は[MIDIメッセージリファレンスマニュアルの「2.2 バンクセレクト / プログラムチェンジ」](manuals/midi-message-reference.md#22-バンクセレクト--プログラムチェンジ)参照)。

```
B0 00 <VoicePatchType>   ; CC#0 バンクセレクトMSB(直接デバイス選択値)
B0 20 <HwBank>            ; CC#32 バンクセレクトLSB(HwBankインデックス)
C0 <HwProg>               ; プログラムチェンジ
```

`VoicePatchType`の値一覧は[同マニュアルの「直接デバイス選択値」表](manuals/midi-message-reference.md#直接デバイス選択値cc0一覧)を参照してください。

### 5.2 編集中のパラメータを送る(プライベートSysEx)

HwPatch/SwPatchのパラメータオーバーライドは、[同マニュアルの「8.1 パッチパラメータオーバーライド」](manuals/midi-message-reference.md#81-パッチパラメータオーバーライドプライベートsysex非標準)で定義したプロトコルをそのまま使います。`target-type=0x00`(チャンネル)を使い、`layer=0`で構いません(単一レイヤーの音色を編集する想定)。

```
F0 00 48 01 <sub-cmd> 00 <ch> 00 <JSONバイト列> F7
```

- `sub-cmd`:HwPatchなら`01`、SwPatchなら`02`
- `<ch>`:このパッチエディタが使っているMIDIチャンネル(0-15)
- JSONは、変更したいフィールドのみを含むオブジェクト([HwPatchリファレンス](manuals/hwpatch-reference.md)/[SwPatchリファレンス](manuals/swpatch-reference.md)参照)

パラメータをつまみで動かすたびに、このSysExを送り直すことで、リアルタイムに音を作り込めます。空オブジェクト`{}`を送ると、オーバーライドが解除され直前のプログラムチェンジで選択した音色本来の値に戻ります。

### 5.3 試聴する

通常のノートオン/オフを送るだけです。

```
9<ch> <note> <velocity>   ; ノートオン
8<ch> <note> <velocity>   ; ノートオフ
```

### 5.4 ネイティブパッチ(ToneLayer)の編集

通常モードの音色(ネイティブパッチ)を編集する場合は、NRPN 97番台([同マニュアルの「3.3 ToneLayerオーバーライド」](manuals/midi-message-reference.md#33-tonelayerオーバーライドnrpn-97))を使います。この場合、あらかじめ通常モード(CC#0=0)でバンク/プログラムを選択しておく必要があります。

### 5.5 後片付け

編集セッションを終える際や、試聴中のノートが残ってしまった場合は、以下を送ってください。

```
B<ch> 78 00   ; CC#120 オールサウンドオフ
B<ch> 79 00   ; CC#121 リセットオールコントローラー(RPN/NRPN/SysExオーバーライドの解除を含む)
```

`121`(リセットオールコントローラー)を送ると、HwPatch/SwPatchオーバーライドも解除されます。また、プログラムチェンジを送るだけでも同様にオーバーライドは解除されます。

---

## 6. 制限事項

- MIDI Out(FITOM_X → パッチエディタ方向)は未実装です。
- 同時接続数は1です。複数のパッチエディタインスタンスを同時に接続することはできません。
- デバイス一覧の取得等、双方向のクエリが必要な機能は、このインターフェースの対象外です(将来的に別途検討)。
